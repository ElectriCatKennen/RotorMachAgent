## 三、技术栈说明

### 3.1 SW COM 接口：pywin32 + comtypes 双轨制

RotorMachAgent 采用 **双轨制** 调用 SOLIDWORKS API：

| 调用方式 | 适用场景 | 典型方法 |
|---------|---------|---------|
| pywin32（IDispatch） | 常规方法 | `InsertSketch`, `CreateLine`, `FeatureRevolve` |
| comtypes（vtable） | 多参数方法（26+参数） | `FeatureCut3`, `FeatureRevolveCut2` |

**原因**：部分 SW API 方法参数过多，pywin32 的 IDispatch 调用会报错，需用 comtypes 的 vtable 方式调用。

### 3.2 数据与代码分离

GB 标准数据存储为独立 JSON 文件：

```
src/data/standards/
├── GB_T1096_平键.json
├── GB_T894_1_弹性挡圈.json
├── GB_T812_圆螺母.json
├── GB_T196_螺纹.json
├── GB_T2822_标准尺寸.json
├── GB_T321_优先数系.json
├── GB_3100_单位换算.json
└── 力学_常数.json
```

**优势**：
- 标准数据可独立维护、扩展
- 支持动态加载不同标准版本
- 代码仅负责读取和使用，不硬编码标准尺寸

### 3.3 计算引擎：scipy + sympy

| 库 | 用途 | 示例 |
|----|------|------|
| `scipy` | 数值计算 | 数值积分、优化求解 |
| `sympy` | 符号计算 | 公式推导、方程求解 |

### 3.4 MCP 协议

MCP（Model Context Protocol）用于 LLM Agent 调用工具：

- 工具封装在 `src/tools/`
- MCP Server 在 `src/agents/mcp_server.py`
- 支持各类 LLM Agent 接入（不绑定具体 LLM）

## FAQ

**Q: 为什么不用 FreeCAD 或 OpenCASCADE？**
A: 目标用户使用 SOLIDWORKS，输出 SW 原生文件可保留特征树，便于二次编辑。FreeCAD 路线会丢失参数化特征。

**Q: comtypes 和 pywin32 冲突吗？**
A: 不冲突。两者互补，pywin32 用于常规调用，comtypes 用于多参数方法。本项目提供桥接函数实现两者协作。

**Q: 是否支持 Linux？**
A: 不支持。SW COM 接口和 pywin32 仅 Windows 可用。

**Q: MCP 协议是什么？**
A: Model Context Protocol，Anthropic 开放的 Agent 工具调用协议。详见 [MCP 官方文档](https://modelcontextprotocol.io/)。
